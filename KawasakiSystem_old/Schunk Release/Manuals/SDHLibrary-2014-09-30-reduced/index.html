Overview of the SDH2 CD

Date: 2014-09-30

This CD contains documents for the SCHUNK dextrous hand SDH. The documents include library source code, documentation files and programs. These various documents are distributed over the subdirectories explained below.

SDH access libraries

./cpp/ SDHLibrary-CPP, release 0.0.2.10, date 2014-02-28

The SDHLibrary-CPP is a C++ library and some demonstration programs to control the SDH from a PC using the C++ programming language.
  • Changelog of the SDHLibrary-C++ release
  • ./cpp/README.txt : directory overview in TXT format
  • ./cpp/doc/index.html : documentation in HTML and PDF format
  • ./cpp/Makefile : to generate the library in Cygwin and Linux environments
  • ./cpp/Makefile-settings : settings for generating the library in Cygwin and Linux environments
  • ./cpp/vcc/SDHLibrary.sln : Visual Studio solutions file to generate the library and demo programs in Windows environments
To compile the C++ Library you might want/have to adjust some settings:
  • In ./cpp/sdh/sdhlibrary_settings.h
    • Set SDH_USE_NAMESPACE to 1 (default) to put all classes in a namespace "SDH". Set to 0 to leave them in the global namespace
  • For Cygwin/Linux: in ./cpp/Makefile-settings
    • Set the LIBRARY_TYPE to static or shared
    • Set the WITH_ESD_CAN to 1 to use the ESD CAN library. Set to 0 if you don't have or don't want to use a CAN adapter from ESD
      • If WITH_ESD_CAN is 1 then adjust the include and library paths for ESD CAN. See EXTRAINCDIRS and EXTRALDFLAGS.
    • Set the WITH_PEAK_CAN to 1 to use the PEAK CAN library. Set to 0 if you don't have or don't want to use a CAN adapter from PEAK
      • If WITH_PEAK_CAN is 1 then adjust the include and library paths for ESD CAN. See EXTRAINCDIRS and EXTRALDFLAGS.
  • For Windows/Visual Studio the settings must be done in the project files:
    • If you have a supported adapter for the CAN-bus from company ESD and want to use it, then:
      • Set the WITH_ESD_CAN preprocessor macro to 1 in order to use the ESD CAN library. Make this setting in the ./cpp/vcc/SDHLibrary.vcproj project file of the SDHLibrary itself (Properties->Configuration Properties->C/C++->Preprocessor->Preprocessordefinitions)
      • adjust the include path for the header file ntcan.h to use ESD CAN, if needed.
      • Add the library file ntcan.lib from its actual location in your environment to the SDHLibrary project ./cpp/vcc/SDHLibrary.vcproj
        (Previously, up to and including SDHLibrary-CPP v0.0.2.0, you had to add the ntcan.lib to all the demo projects)
    • If you have a supported adapter for the CAN-bus from company PEAK and want to use it, then:
      • Set the WITH_PEAK_CAN preprocessor macro to 1 in order to use the PEAK CAN library. Make this setting in the ./cpp/vcc/SDHLibrary.vcproj project file of the SDHLibrary itself (Properties->Configuration Properties->C/C++->Preprocessor->Preprocessordefinitions)
      • adjust the include path for the header file Pcan_usb.h to use PEAK CAN, if needed.
      • Add the library file pcan.lib from its actual location in your environment to the SDHLibrary project ./cpp/vcc/SDHLibrary.vcproj
        (Previously, up to and including SDHLibrary-CPP v0.0.2.0, you had to add the pcan.lib to all the demo projects)
      • Note: The SDHLibrary currently supports the PCAN-Light driver API from PEAK only, not the newer PCAN-Basic variant. On linux please use the "chardev" driver, not the "netdev" one.
    • If you do not have a supported ESD and or PEAK CAN card or do not want to use it, then:
      • Set the WITH_ESD_CAN and/or WITH_PEAK_CAN preprocessor macro to 0. Make this setting in the ./cpp/vcc/SDHLibrary.vcproj project file of the SDHLibrary itself (Properties->Configuration Properties->C/C++->Preprocessor->Preprocessordefinitions)
      • Disable or remove the files canserial-esd.h and canserial-esd.cpp and/or canserial-peak.h and canserial-peak.cpp from the project file of the library: ./cpp/vcc/SDHLibrary.vcproj
      • Remove the the ntcan.lib and/or pcan.lib, if it is present in the SDHLibrary project
        (Previously, up to and including SDHLibrary-CPP v0.0.2.0, you had to remove the ntcan.lib / pcan.lib files from all the demo projects)

./python/ SDHLibrary-python, release 0.0.2.9, date 2014-09-30

The SDHLibrary-python is a Python library (package) and some demonstration scripts to control the SDH from a PC using the Python scripting language.
  • Changelog of the SDHLibrary-Python release
  • ./python/README.txt : directory overview in TXT format
  • ./python/doc/index.html documentation in HTML and PDF format
Windows:
  • ./python/dist/SDHLibrary-python-0.0.2.9.win32.exe Windows installer with the SDH python library and some demonstration scripts.

    On Windows you might have to install some other packages first. These are contained on the CD in the ./python/misc/ directory, or can be retrieved from the internet. There are two supported versions of the python environment available, v2.5 and v2.6. Some of the packages are version specific, please use either a python 2.6 or python 2.5 environment.
    Remark: CAN support for python is available only for python 2.5 and 2.6 by ESD!

    The packages must be installed in the order given below (top to bottom).
    • Python 2.7 specific packages:
      • ./python/misc/python-2.7.5.msi The Python scripting language for windows itself
        (also available online: http://www.python.org/)
      • ./python/misc/pywin32-218.win32-py2.7.exe Python windows compatibility package (stdlib)
        (also available online: http://sourceforge.net/projects/pywin32/)
      • ./python/misc/Readline-1.7.win32-py2.7.exe Python package for emulating readline
        (also available online: http://newcenturycomputers.net/projects/)
    • Python 2.6 specific packages:
      • ./python/misc/python-2.6.1.msi The Python scripting language for windows itself
        (also available online: http://www.python.org/)
      • ./python/misc/pywin32-212.win32-py2.6.exe Python windows compatibility package (stdlib)
        (also available online: http://sourceforge.net/projects/pywin32/)
      • ./python/misc/Readline-1.7.win32-py2.6.exe Python package for emulating readline
        (also available online: http://newcenturycomputers.net/projects/)
    • Python 2.5 specific packages:
      • ./python/misc/python-2.5.1.msi The Python scripting language for windows itself
        (also available online: http://www.python.org/)
      • ./python/misc/pywin32-212.win32-py2.5.exe Python windows compatibility package (stdlib)
        (also available online: http://sourceforge.net/projects/pywin32/)
      • ./python/misc/Readline-1.7.win32-py2.5.exe Python package for emulating readline
        (also available online: http://newcenturycomputers.net/projects/)
  • ./python/misc/pyserial-2.7.win32.exe Python package for accessing the serial interface
    (also available online: http://sourceforge.net/projects/pyserial/)
Linux / Cygwin:
  • ./python/dist/SDHLibrary-python-0.0.2.9.cygwin-1.7.31-i686.tar.gz Python package a compressed tar archive containing the SDH python library and some demonstration scripts. These files are also contained uncompressed in the directories ./python/sdh/ and ./python/demo/ respectively.

    For Linux you might have to install some other packages first. These are partly contained on the CD in the ./python/misc directory, are available via your favourite package installation tool or can be retrieved from the internet.
    • ./python/misc/pyserial-2.7.tar.gz Python package for accessing the serial interface
      (or look for something like python-serial or pyserial in the package installer,
      or online: http://sourceforge.net/projects/pyserial/)
    • Tkinter: Python package for GUI based user interaction
      (or look for something like tkinter python-tk in the package installer,
      or online: http://wiki.python.org/moin/TkInter)

    On Linux you might have to convert the line endings from windows style (CR,LF) to unix style (LF) to make the demo scripts work. To do this just call 'make dos2unix' in the ./python/ directory.

    In order to use the python package sdh you have to make it known to the python interpreter. You can do this by installing it to some standard location, or by modifiying the PYTHONPATH environment variable. To install to the standard location call python setup.py install in the ./python/ directory. To modify the PYTHONPATH you can use the provided ./sdhsetenv script. This must be 'sourced in', e.g. in bash use source sdhsetenv in the distributions top level directory ./.

./firmware/ Firmware

The firmware is the software that runs within the SDH itself and which can be updated too.
  • ./firmware/sdhflash-*.cygwin-*.exe or a ./firmware/sdhflash-*.win32.exe Windows programs that install program and documentation to update the firmware of the SDH
  • ./firmware/sdhflash-*.tar.bz2 Compressed archives with program and documentation to update the firmware of the SDH on Linux
  • ./firmware/Firmware_DSACON32-M*.zip Compressed archives with firmware for the tactile sensor controller DSACON32m within the SDH
  • http://www.weiss-robotics.de/en/support/downloads.html The latest firmware for the tactile sensor controller DSACON32m, available online from WEISS-Robotics
A description of the update procedure for both the SDH itself and the DSACON32m is given here: ./doc/SDH2_configuration-and-update.pdf.

./CAD/ CAD data

The CAD data or 3D models of the SDH components are contained in the directory ./CAD. The data is contained in the formats IGES, STEP and Wavefront. A more detailed explanation is contained in ./CAD/position of the central points.pdf
